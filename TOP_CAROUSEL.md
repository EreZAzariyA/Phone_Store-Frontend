# TopCarousel — Frontend Integration Guide

This document describes the `TopCarousel` feature on the backend and how a frontend agent should integrate with it.

---

## What It Is

The TopCarousel is a homepage hero/banner slider. Each item represents one slide with a background image, optional text overlay, and an optional call-to-action link.

---

## Data Model

Each carousel item has the following fields:

| Field          | Type      | Required | Notes                                      |
|----------------|-----------|----------|--------------------------------------------|
| `id`           | `number`  | yes      | Auto-generated by the backend              |
| `displayOrder` | `number`  | yes      | Controls slide order (ascending)           |
| `imageURL`     | `string`  | yes      | Full URL to the slide background image     |
| `isActive`     | `boolean` | yes      | Only active slides should be rendered      |
| `title`        | `string?` | no       | Main headline text                         |
| `subtitle`     | `string?` | no       | Supporting body text                       |
| `eyebrow`      | `string?` | no       | Small label shown above the title          |
| `linkURL`      | `string?` | no       | Where the CTA button navigates             |
| `linkText`     | `string?` | no       | Label for the CTA button                   |

---

## API Endpoints

Base path: `/api/top-carousel`

### GET `/api/top-carousel`
- **Auth**: Public (no token required)
- **Response**: Array of all carousel items, sorted by `displayOrder` ascending
- **Use this to**: Fetch slides for the homepage carousel

```ts
const res = await fetch('/api/top-carousel');
const slides = await res.json();
// Filter to only active ones on the client:
const activeSlides = slides.filter(s => s.isActive);
```

### POST `/api/top-carousel`
- **Auth**: Admin JWT required (`Authorization: Bearer <token>`)
- **Body**: `AddTopCarouselItemRequest`

```json
{
  "displayOrder": 1,
  "imageURL": "https://example.com/image.jpg",
  "isActive": true,
  "title": "New iPhone",
  "subtitle": "Discover the future",
  "eyebrow": "Just Arrived",
  "linkURL": "/phones/123",
  "linkText": "Shop Now"
}
```

### PUT `/api/top-carousel`
- **Auth**: Admin JWT required
- **Body**: `UpdateTopCarouselItemRequest` — same fields as POST plus `id`

```json
{
  "id": 3,
  "displayOrder": 2,
  "imageURL": "https://example.com/updated.jpg",
  "isActive": false,
  "title": null,
  "subtitle": null,
  "eyebrow": null,
  "linkURL": null,
  "linkText": null
}
```

### DELETE `/api/top-carousel/{id}`
- **Auth**: Admin JWT required
- **Example**: `DELETE /api/top-carousel/3`
- **Response**: `200 OK` (empty body)

---

## Frontend Rendering Rules

1. **Filter by `isActive`**: The API returns all items. Only render slides where `isActive === true`.
2. **Sort by `displayOrder`**: The API already returns items sorted, but rely on this field if you re-sort client-side.
3. **Optional fields**: `title`, `subtitle`, `eyebrow`, `linkURL`, and `linkText` can be `null` or empty — render them only when present.
4. **CTA button**: Render the link button only if both `linkURL` and `linkText` are non-empty.
5. **Eyebrow**: Shown as a small uppercase tag/label above the `title`.

---

## Suggested Component Structure

```
<Carousel>
  {activeSlides.map(slide => (
    <Slide key={slide.id} backgroundImage={slide.imageURL}>
      {slide.eyebrow   && <Eyebrow>{slide.eyebrow}</Eyebrow>}
      {slide.title     && <Title>{slide.title}</Title>}
      {slide.subtitle  && <Subtitle>{slide.subtitle}</Subtitle>}
      {slide.linkURL && slide.linkText && (
        <CTAButton href={slide.linkURL}>{slide.linkText}</CTAButton>
      )}
    </Slide>
  ))}
</Carousel>
```

---

## Admin UI Notes

The admin panel should provide a CRUD interface for carousel items. Key behaviors:

- **Create**: POST with all required fields; `isActive` defaults to `false` on the backend entity but the admin should let the user set it explicitly.
- **Edit**: PUT with the full item (all fields, not a partial patch).
- **Delete**: DELETE by id; confirm before sending.
- **Reorder**: Change `displayOrder` values and PUT each modified item.

```ts
const token = localStorage.getItem('token'); // or from auth context
const headers = {
  'Content-Type': 'application/json',
```
